## 1. OddFi 是什么

OddFi 是部署在 BNB Smart Chain（BSC）上的去中心化体育预测市场。连接钱包即可使用，无需注册、无需邮箱、无需 KYC，资金由用户自托管。

| 项目 | 内容 |
|---|---|
| 所在链 | BNB Smart Chain（BSC，chainId = 56） |
| 平台代币 | ODD（BEP-20，总量 1 亿枚，不增发） |
| 登录方式 | 钱包连接，无需 KYC |
| 结算方式 | 双源核实 + 多签确认，链上执行 |
| 支持充值 | 多种链（暂不支持 TRX/Tron） |

### 为什么选择 OddFi

传统预测市场存在三个结构性问题，OddFi 从底层重构了资金模型与收益分配：

| 痛点 | 传统方案 | OddFi 方案 |
|---|---|---|
| 滑点严重 | AMM 定价，大额预测滑点高 | Betting Pool 统一对手方，固定回报倍率接单 |
| 资金闲置 | 待结算资金完全锁死 | 仓位借贷：用待结算仓位抵押，借出额度继续参与 |
| 收益垄断 | 平台独占全部利润 | 任何人可向 Betting Pool 注资，按份额分享平台 Margin |
| 代币无锚定 | 代币与业务脱钩 | ODD 买卖税机制让代币价值与平台业务深度挂钩 |

### 与传统中心化体育预测平台的对比

| 维度 | OddFi | 传统平台 |
|---|---|---|
| 账户 | 仅需钱包连接 | 邮箱 + KYC + 身份验证 |
| 资金托管 | 智能合约托管，平台无法单方面提取 | 平台持有你的资金 |
| 结算 | 链上可验证 | 信任运营方 |
| 透明度 | 回报倍率推导过程公开，Margin 结构透明 | 定价是黑箱 |
| 领取 | ≤ $5,000 即时到账 | 人工审核，1–5 天 |
| 收益分配 | 做市用户共享 Margin | 收益归平台 |

## 2. 快速开始

### 第一步：连接钱包

OddFi 支持所有兼容 WalletConnect 的主流 EVM 钱包：MetaMask、OKX Wallet、Bitget Wallet、Phantom、Binance Wallet、Coinbase Wallet 等。

**步骤：**
1. 点击页面右上角「连接钱包」
2. 选择钱包类型
3. 批准连接请求
4. 签署认证消息
5. 完成登录

> 钱包须切换至 BSC（chainId = 56）。首次签名后自动注册，回访用户通过地址识别。
>
> 移动端：已安装钱包 App 则直接打开授权；未安装则跳转下载钱包页面。

### 第二步：获取 ODD

ODD 是 OddFi 平台的唯一计价与结算代币，下注前需先持有 ODD。

获取方式：

| 方式 | 说明 |
|---|---|
| PancakeSwap 购买 | 在 PancakeSwap 通过 ODD/USDT 交易对买入 ODD |
| 跨链兑换 | 通过 Relay 跨链桥，将其他 EVM 链上的资产兑换为 ODD |
| 同链代币兑换 | 在 PancakeSwap 将 BSC 上的其他代币（如 BNB、USDT）兑换为 ODD |

> 购买 ODD 时会触发 5% 买入税，作为推荐人质押奖励（详见[邀请奖励](#8-邀请奖励)）。

### 第三步：充值到账户

进入「钱包」→「充值」，复制你的个人代理钱包地址，将 ODD 发送至该地址，12 个区块确认后（约 36 秒）余额自动到账。

详细充值说明见[充值与提现](#4-充值与提现)。

### 第四步：开始预测

余额到账后即可在赛事列表选择比赛，选择盘口和方向，输入金额确认下注。

## 3. 世界杯玩法

### 盘口类型

OddFi 目前支持足球的三种核心盘口：

| 盘口 | 全称 | 说明 |
|---|---|---|
| 1X2 | 胜平负 | 预测主胜 / 平局 / 客胜，最基础的玩法 |
| AH | 让球盘（Asian Handicap） | 给一方加减球数后判定胜负，消除平局 |
| OU | 大小球（Over/Under） | 预测总进球数高于或低于指定值 |

### 胜负预测（1X2）

预测 90 分钟常规时间的比赛结果（不含加时赛和点球）。

示例：皇家马德里 vs 巴塞罗那

| 选项 | 回报倍率 | 投入 100 ODD 的预期回报 |
|---|---|---|
| 1（皇马胜） | 2.4 | 240 ODD |
| X（平局） | 3.2 | 320 ODD |
| 2（巴萨胜） | 2.85 | 285 ODD |

> 回报倍率越低 = 市场认为越可能发生。

### 让分预测（Asian Handicap）

给强队一个虚拟球数劣势，缩小或消除平局选项。

OddFi 提供三个让球盘口：-0.5、-1、-1.5。

示例：曼城 -1.5 vs 阿森纳 +1.5

| 比赛结果 | 买曼城 -1.5 | 买阿森纳 +1.5 |
|---|---|---|
| 曼城 3:1（净胜 2 球） | 赢 | 输 |
| 曼城 2:1（净胜 1 球） | 输 | 赢 |
| 1:1 平局 | 输 | 赢 |

> 理解技巧：在最终比分上给曼城 -1.5 球，调整后仍领先就赢。

让分速查表：

| 盘口 | 强队需要... | 弱队赢的条件 |
|---|---|---|
| -0.5 | 赢 1 球以上 | 平或赢 |
| -1 | 赢 2 球以上（赢 1 球退款） | 平或赢（输 1 球退款） |
| -1.5 | 赢 2 球以上 | 赢、平或仅输 1 球 |

### 总分预测（Over/Under）

预测双方总进球数高于（Over/大）还是低于（Under/小）指定盘口线。

半球线（1.5、2.5、3.5）— 只有赢和输。

示例：盘口线 2.5

| 比赛比分 | 总进球 | 大 2.5 | 小 2.5 |
|---|---|---|---|
| 2:1 | 3 | 赢 | 输 |
| 1:1 | 2 | 输 | 赢 |

总分速查表：

| 盘口线 | 大（Over）赢的条件 | 小（Under）赢的条件 |
|---|---|---|
| 1.5 | ≥ 2 球 | ≤ 1 球 |
| 2.5 | ≥ 3 球 | ≤ 2 球 |
| 3.5 | ≥ 4 球 | ≤ 3 球 |

### 赔率机制

#### 赔率怎么来的

平台从第三方专业赔率数据源获取参考倍率，去除原始 Margin 后还原真实概率，再加入平台 10% Margin 生成开盘倍率。

```
回报倍率 = 1 / （真实概率 × Overround 因子）
```

典型 Overround：110%。

#### 赔率怎么读

| 赔率 | 隐含概率 | 含义 |
|---|---|---|
| 1.5 | 66.70% | 强势热门 — 低回报 |
| 2.0 | 50.00% | 五五开 — 2 倍投入金额 |
| 3.0 | 33.30% | 冷门方 — 3 倍投入金额 |
| 5.0 | 20.00% | 大冷门 — 5 倍投入金额 |

#### 动态调整

每笔预测后，系统自动检测各方向的累计金额差异并动态调整：

- **重仓方向**：倍率压低（降低平台风险）
- **轻仓方向**：倍率上浮（吸引反向预测，平衡敞口）

倍率在开赛前 1 小时锁定，此后不再变化。变动幅度超过 0.05 时实时推送至前端。

#### 倍率变动确认

从填写金额到确认期间倍率变动超过 0.05，系统弹窗提示最新倍率，需重新确认。

### 下注流程

1. 在赛事列表选择比赛
2. 点击对应赔率（方向 + 盘口）
3. 在注单栏输入下注金额
4. 系统校验余额、限额、倍率变动
5. 确认下注 → 调用合约锁定资金
6. 赛事结算后自动结算待领取收益

> **赔付计算：潜在赔付 = 下注金额 × 赔率**
>
> 示例：下注 100 ODD，赔率 2.78 → 潜在赔付 = 278 ODD。

### 结算

赛事结束后，后端从两个独立数据源核实比赛结果：

- **两源一致** → 生成多签 calldata，Admin 钱包确认 → 链上结算
- **两源不一致** → 进入人工核查流程

结算后：赢了 → 赔付计入可领取余额；输了 → 扣除下注金额。

### 领取奖励

1. 前往 Portfolio → 领取区域
2. 查看可领取金额和明细
3. 点击「领取」→ 签署钱包交易（你支付 Gas 费）
4. 资金转入 Cash 余额

> 领取金额扣除 10% 平台手续费。示例：可领取 100 ODD → 手续费 10 ODD → 你收到 90 ODD。

### 下注状态说明

| 状态 | 含义 |
|---|---|
| Confirmed | 链上已确认 |
| Cancel | 已取消，资金已释放 |
| Won | 命中，未领取奖励 |
| Lost | 未命中 |
| Claim | 待领取奖励 |

### 串关（Multi）

将多场比赛的预测组合成一张注单，所有场次全部命中才算赢。

| 规则项 | 说明 |
|---|---|
| 腿数 | 最多 10 个赛事 |
| 跨赛事 | 同一场比赛只能出现一次，不能同时提交多个玩法的多个方向 |
| 倍率计算 | 各腿倍率连乘（锁定预测时的倍率） |
| 最大回报 | $10,000（超出则拒绝接单） |
| 任意一腿未命中 | 整张组合订单判负 |

> 组合预测仓位不可用于仓位借贷的抵押。
>
> 示例：3 场组合，各腿倍率 1.80 / 2.10 / 1.95 → 组合倍率 = 7.37 → 投入 $100，全部命中可获 $737。

### 提前结算（Cancel）

| 项目 | 说明 |
|---|---|
| 可用条件 | 仅限活跃预测，赛事结算前 |
| 计算方式 | 基于当前实时倍率 |
| 手续费 | 从兑现金额中扣除 |
| 结算 | 即时到账，资金转入 Cash 余额 |

### 下注限额

| 规则 | 说明 |
|---|---|
| 最低下注 | 1 ODD |
| 单笔上限 | 赔率 < 2.0：$800 / 赔率 2.0–3.5：$500 / 赔率 > 3.5：$300 |
| 用户累计未平仓上限 | $15,000 |
| 赔率格式 | 小数赔率（欧洲格式） |
| 余额不足 | 提示「余额不足」并提供充值入口 |

## 4. 充值与提现

### Portfolio（资产总览）

资金到账后进入你的 Portfolio，分四个口袋：

| 口袋 | 说明 |
|---|---|
| Cash | 可用余额，用于预测或提现 |
| Bet | 锁定在活跃预测中的资金（未结算） |
| Stake | 锁定在质押中的资金 |
| Liquidity | 在做市池中的资金 |

所有金额以 ODD + USD 等值显示。

---

### 充值

OddFi 为每个用户分配独立的**个人代理钱包**，充值时将资金转入该地址，平台通过该地址识别用户身份，无论从哪个钱包转入都能正确归账。

支持三种充值方式：

| 方式 | 说明 |
|---|---|
| 直接转账 | 将 ODD（BSC BEP-20）直接发送至你的代理钱包地址 |
| 同链兑换 | 在 PancakeSwap 将 BSC 上的 BNB 或其他代币兑换为 ODD，再转至代理钱包 |
| 跨链兑换 | 通过 Relay 跨链桥，将其他 EVM 链上的资产兑换为 ODD |

**直接转账步骤：**

1. 进入「钱包」→「充值」
2. 复制个人代理钱包地址
3. 从你的 EOA 钱包发送 ODD（BSC 链 BEP-20）至该地址
4. 等待 12 个区块确认（约 36 秒）
5. 余额自动到账

> 仅支持 BSC 链上的 ODD，请勿从以太坊主网或其他链直接转账，否则资产无法找回。

---

### 提现

进入「资产管理」→「提取」：

- **≤ $5,000**：自动处理，链上即时到账，无需操作
- **> $5,000**：进入人工审核，24 小时内完成

可提现到任何有效的 EVM 钱包地址，不限于连接时使用的地址。收到 ODD 后，可前往 PancakeSwap 通过 ODD/USDT 交易对兑换为 USDT 或其他代币。

## 5. 添加流动性（做市）

向 Betting Pool 注入 ODD，成为平台的对手方流动性提供者。用户预测命中由 Betting Pool 支付，未命中归入 Betting Pool，做市用户按份额分享平台整体盈利。

### 收益构成

| 版本 | 收益来源 | LP 分成 | 平台 + 质押池 |
|---|---|---|---|
| ODD 版 | 平台整体 Margin + ODD 卖出利润税 4pp | 80% | 10% + 10% |

收益按「你的存入金额 / 池子总金额」比例分配。

### 锁定期与收益乘数

| 锁定期 | 收益乘数 |
|---|---|
| 30 天 | 1.0x |
| 60 天 | 1.8x |
| 90 天 | 4.0x |

### 赎回规则

- **到期赎回**：本金 100% 返还，收益正常领取
- **提前赎回**：本金 100% 返还，**收益 100% 罚没**

### 风险提示

做市用户承担平台整体亏损风险。若某段时期用户预测总命中率偏高，Betting Pool 净流出，做市用户可能面临阶段性收益为负。平台通过动态倍率和信号灯系统控制单盘敞口。

## 6. ODD 质押

锁定 ODD 代币，分享平台税收收益，同时解锁更高的每日预测限额（VIP 等级）。

### 收益来源

| 来源 | 说明 |
|---|---|
| ODD 卖出利润税 10% | 每笔有盈利的 ODD 卖出自动注入 |
| 推荐人无效时的买入税 5% | 推荐人未质押或无推荐人时转入 |
| 提前赎回罚没收益 | 用户提前赎回时，累计收益 100% 转回奖励池 |

收益按「用户质押量 × 乘数 / 全网质押量 × 乘数」权重分配。

### 锁定期与收益乘数

| 锁定期 | 收益乘数 | 备注 |
|---|---|---|
| 7 天 | 0.5x | 推荐人奖励自动质押专用，不可手动选择 |
| 30 天 | 1.0x | — |
| 60 天 | 1.8x | — |
| 90 天 | 4.0x | — |

### 赎回规则

- **到期赎回**：本金 100% 返还，收益正常领取
- **提前赎回**：本金 100% 返还，**收益罚没**

### 质押与下注联动

| 累计未平仓金额 | 质押要求 |
|---|---|
| ≤ $500 | 无需质押，直接预测 |
| > $500 | 最大未平仓 ≤ 当前有效质押量（ODD 计价） |

## 7. 仓位借贷

以一个待结算的下注仓位作为抵押，借出额度用于再次下注。适合手中有进行中仓位但暂时没有额外资金的用户。

```
可借额度 = 抵押仓位本金 × 50%
```

示例：抵押一个 $200 的待结算订单，可借出最多 $100。

### 使用限制

| 限制项 | 规则 |
|---|---|
| 仓位归属 | 必须是本人的仓位 |
| 仓位状态 | 必须处于 Pending（待结算） |
| 仅限单注 | 组合预测不可作为抵押 |
| 禁止连环借贷 | 借贷产生的仓位不可再次抵押 |
| 时间窗口 | 借贷仓位比赛结束时间须 ≤ 抵押仓位比赛结束时间 + 7 天 |

### 结算逻辑

| 抵押仓位 | 借贷仓位 | 结果 |
|---|---|---|
| 命中 | 命中 | 借贷仓位正常发放奖励，全额收取 |
| 命中 | 未命中 | 借贷仓位正常判负，抵押仓位照常发放奖励 |
| 未命中 | 命中 | **借贷仓位赢利归平台，用户无法提取** |
| 未命中 | 未命中 | 借贷仓位正常取消，无额外损失 |

> 核心逻辑：抵押仓位若未命中，借贷仓位的一切收益归平台所有，以此覆盖借贷风险。

## 8. 邀请奖励

连接钱包后，每个用户自动获得唯一邀请链接。邀请关系链上永久绑定，一旦绑定不可更改。

### 收益来源

| 收益类型 | 规则 |
|---|---|
| 直推返佣 | 被邀请人每笔有效预测 Margin 的 5%，实时结算 |
| 买入税奖励 | 被邀请人在 DEX 买入 ODD 时，买入金额的 5% 以 ODD 形式为你质押（7 天锁定） |

> 推荐人须处于有效质押状态才能获得买入税奖励。未满足时，该 5% 转入 StakingPool。

### 团队级差奖励

| 等级 | 团队累计预测 | 团队返佣加成 |
|---|---|---|
| — | < $10,000 | 仅直推 5%，无加成 |

| S3 | ≥ $100,000 | +0.12 |

| S5 | ≥ $1,000,000 | +0.20 |

> 采用级差制：上级只拿差额，不重复抽取下级全部收益。

### 首充奖励

| 首充金额 | 被邀请人获得 | 邀请人获得 |

| ≥ $50 | 10 ODD | 5 ODD |

### 买卖税

#### 买入税：5%

用户在 PancakeSwap 买入 ODD 时触发，全部作为推荐人质押奖励（自动质押 7 天）。推荐人未质押或无推荐人时转入 StakingPool。

#### 卖出利润税：25%（仅对盈利部分）

```
卖出利润 = 卖出金额 − （卖出数量 × 持仓均价）
亏损卖出：不征税
盈利卖出：对利润部分收取 25%
```

税收流向：

| 去向 | 比例（占利润税） | 说明 |
|---|---|---|
| Token Pool（PancakeSwap LP） | 5pp | 自动注入 ODD/USDT 交易对，LP Token 永久锁仓 |
| StakingPool 奖励池 | 9pp | 分配给所有活跃质押用户 |
| 平台收益 | 6pp | 转入平台地址 |
| Betting Pool | 4pp | 自动补充对赌池资金 |
| 销毁 | 1pp | 转入 `0xdead` 地址，永久销毁 |

### 增长飞轮

```
更多用户下注
  ↓
ODD 需求增加 → DEX 买入 → 推荐人获奖励 → 推荐更多用户
  ↓
ODD 价格上涨 → 持仓产生利润 → 卖出缴纳利润税
  ↓
质押池 / Betting Pool / Token Pool 持续获得税收注入
  ↓
Betting Pool 扩大 → 可支撑更大投注 → 吸引大额用户
  ↓
平台 Margin 增加 → LP 收益提升 → 吸引更多做市流动性
```

### 通缩机制

- **卖出利润税 1pp**：每笔盈利卖出自动销毁至 `0xdead`
- **季度回购销毁**：基于平台收入定期回购 ODD 销毁
- **特殊事件销毁**：如预测池结余销毁

## 10. 费用与安全

### 全平台手续费总表

| 操作 | 手续费 | 支付方 |
|---|---|---|
| 充值（直接转账） | Gas fee | 用户 |
| 预测 | 无 | — |
| 提前结算 | 无 | — |
| 领取奖金 | 10% 平台手续费 + Gas fee | 从领取金额中扣除 |
| 买入 ODD | 5% 买入税 | 自动扣除 |
| 卖出 ODD（盈利部分） | 25% 利润税 | 自动扣除 |
| Gas 费（所有链上操作） | 链上 Gas fee | 用户 |
| 充值/提现（兑换路径） | DEX 标准费率 + Gas | 用户 |

### 安全措施

| 措施 | 说明 |
|---|---|
| 资金托管 | 所有用户资金由用户钱包自托管，平台无法单方面提取 |
| 多签结算 | 赛事结算需要 Gnosis Safe 多签钱包审批 |
| ReentrancyGuard | 所有资金操作防重入攻击 |
| Pausable | 紧急情况下可暂停合约，保护用户资金 |
| 交易冷却 | 同一地址 1 分钟交易冷却，防 MEV 攻击 |
| 单笔卖出上限 | 单次卖出 ODD 不超过持仓的 10%，防砸盘 |
| 第三方审计 | 主网部署前强制完成安全审计（CertiK / SlowMist / PeckShield） |
| LP 锁定 | DEX 流动性池代币至少锁定 1 个月 |
| 价格操纵防护 | TWAP / 预言机交叉验证，防范闪电贷攻击 |
| 余额校验 | 所有操作前端 + 合约双重余额校验 |

### 风险提示

- **智能合约风险**：虽经审计，合约仍可能存在未发现漏洞，请勿投入超过承受范围的资金
- **市场风险**：ODD 价格受市场影响，做市和质押收益以 ODD 计价时存在价格波动风险
- **流动性风险**：做市用户资金处于锁定期间无法提前无损提取
- **赛事风险**：极端情况下（赛事取消、争议结果）可能触发人工结算流程，结算时间延后

## 11. 常见问题

### 账户与连接

| 问题 | 回答 |
|---|---|
| 需要创建账户吗？ | 不需要。连接 EVM 钱包并签署消息即可，无需邮箱、密码、KYC |
| 支持哪些钱包？ | MetaMask、OKX Wallet、Bitget Wallet、Phantom、Binance Wallet、Coinbase Wallet，及所有兼容 WalletConnect 的钱包 |
| OddFi 运行在哪条链？ | BNB Smart Chain（BSC，chainId = 56） |
| 需要 BNB 吗？ | 需要，用于支付 Gas 费，钱包中需保持少量 BNB |
| 钱包切换了账户怎么办？ | OddFi 检测到账户变更会提示使用新地址重新认证 |
| 有移动端 App 吗？ | OddFi 是 Web 应用，桌面端和移动端均优化。移动端建议使用系统浏览器 |

### 充值与提现

| 问题 | 回答 |
|---|---|
| 充值到账了但余额为 0？ | 充值需要 12 个区块确认（约 36 秒）。超过 5 分钟仍未到账，请确认是否为 BSC 链上的 ODD，以及是否转入了正确的代理钱包地址 |
| 可以提现到其他钱包吗？ | 可以。可提现到任何有效的 EVM 钱包地址 |
| 为什么我的充值地址和登录钱包地址不同？ | OddFi 为每位用户分配独立的代理钱包作为充值地址，方便从任意钱包充值时平台都能识别归属 |

### Bet 与回报

| 问题 | 回答 |
|---|---|
| 回报倍率在确认前变了怎么办？ | 变动超过 0.05 时系统弹窗提示，需重新确认。锁定倍率以确认时为准 |
| 有最低预测额吗？ | 最低 1 ODD |
| 如何领取奖金？ | 赛事结算后在 Portfolio 领取区域点击「领取」签署交易。领取时扣除 10% 手续费 |
| 组合预测中有一场没结果，其他都赢？ | 组合预测需所有腿全部结算完毕才计算结果。任意一腿未结算，整张组合预测保持 Pending |

### 仓位借贷

| 问题 | 回答 |
|---|---|
| 抵押仓位赢了但借贷仓位输了？ | 借贷仓位正常判负，损失借贷金额（最多为抵押本金 50%）。抵押仓位赢正常发放奖励 |

### 质押与做市

| 问题 | 回答 |
|---|---|
